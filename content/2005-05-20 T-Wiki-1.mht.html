C/C++ unit testing from Python with ctypes

The C/C++ unit testing frameworks that I've used require lots and = lots of=20 boilerplate and are complex to use. Enter Python and ctypes.=20

  • Python unit testing frameworks make it very easy to deal with test = suites=20 and test cases. Python in general makes setting up the test conditions = (e.g.=20 launching a process) and validating them (e.g. parsing a string) much = simpler.=20
  • ctypes makes it very easy to call entry points for a dll. =

So, the simplest way I've found to have C/C++ unit tests is to code a = dll=20 with the entry points that you'll be calling from the python code (you = may not=20 even need to write a testing dll if you are testing a dll that provides = good=20 enough entry points).=20

I just started this path and I'm very happy with it (the only thing = that=20 annoys me is the amount of effort that we could have saved if we had = started=20 before :-). I'm dealing now with a small test suite that can use the = regular dll=20 entry points; however, if more test code needs to be in the C/C++ side = (e.g. you=20 want to inspect objects), maybe it would be worth to have have some = script to=20 generate the python code that calls the entry points and the .def file = that=20 exports them.=20

BTW: I've just seen that ctypes has now an experimental code= =20 generator that = converts=20 declarations in C header files into executable Python code: enums, = structs,=20 unions, function declarations, com interfaces, and preprocessor = definitions.=20
