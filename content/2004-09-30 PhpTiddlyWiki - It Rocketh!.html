This is a bit of a hack. In an ideal world, clicking the save button should just give you a file save dialogue box and let you choose where to save your spanking new personal TiddlyWiki. Unfortunately doing stuff in web browsers is never that easy, and there's a couple of hoops to be jumped through. See below for a quick guide.



The steps to save your changes as a new, standalone TiddlyWiki are simple, but can be error prone.

1. Make sure that all the text is selected in the edit box above. Copy it to the clipboard.
2. Go back to the browser window showing your editted TiddlyWiki and save the HTML as a new file.
3. Open the HTML file in a text editor like Notepad. Scroll to the bottom and locate the marker lines picked out with a row of asterisks.
4. Select the text from just above that marker back up to the previous marker.
5. Paste the new text in.
6. Save the HTML file.
Suggestions or improvements welcome.

I tried this on a local copy, and it seems to work. The idea is to have the browser's "save" function save a functioning TiddlyWiki (client version, not server version, but this is the only TiddlyWiki I can save changes to online to post this). I have only tested on FireFox .9.2 for windows. - To the function refreshAll(), I added a call to a new function writeBody() to the very first line - The new function writeBody() looks for a div called "layoutArea", clears it, and into it copies the contents of a div called "siteLayout" - There is a style setting "siteLayout" to "display:none" - The portion of the main html from before div id="copyright" to after the license text is enclosed in a div called "layoutArea" - After the "saveMessage" div is a div called "siteLayout" that has the html for layout before the scripts run their magic. The modification simply blows away the contents of the new div "layoutArea" and copies the contents of "siteLayout" into it whenever "refreshAll" is called. Seems to be working for me, though I certainly can't guarantee that it is bug free. UPDATE: While this works well for Firefox, it apparently doesn't do crap for Safari, which doesn't save the changed DOM at all, it always saves the original source.
